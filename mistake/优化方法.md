只能单角度的话，想把精度再抬上去，核心思路是：**把信息“榨干”+ 强先验约束 + 让 loss 聚焦“可辨识方向”**。下面给你一套“能立刻改、能看到提升”的做法（都基于你当前代码结构）。

# A. 训练目标：不只盯“系数 MSE”，再加投影一致性 + 先验

你现在的反传只用系数域 MSE：
 `loss = MSE(coeff_pred, coeff_true)`（见 `train.py`）。
 单角度下很多方向不可辨识，单盯系数会卡平台。建议把 loss 改成多项组合：

1. **投影域一致性**（强制 F 一致）：

```python
F_pred = self.model.optimizer.operator(coeff_pred)   # Φ G c_pred
loss_proj = torch.mean(torch.abs(F_pred - F_observed.squeeze(1))**2)
```

> 这样即使某些系数难直接逼近，网络也会尽可能把**观测域**对齐（你现在只是把它当指标，而不是训练信号）。`operator` 就在 `TheoreticalCTNet` 里用到，能直接拿来算 F（见 `model.py`）。

1. **图像域重建误差**（把不可辨识的系数误差“摊平”）：
    如果你已有从系数→图像的映射（`CoefficientMapping`/B样条），在 `train()` 里把 `coeff_pred` 先重建图像 `f_pred`，再加：

```python
loss_img = torch.mean((f_pred - f_true)**2)
```

> 你的数据生成里已有 B 样条生成器和 `CoefficientMapping`（同一个类名出现在 `model.py/train.py`），可以用它们重建图像后再算 MSE。 

1. **TV/Dirichlet 先验**（单角度必须靠先验）：
    你在理论梯度里已有 Dirichlet/TV 的计算，直接把**显式正则**并进 loss（图像域更直观）：

```python
# 简易各向同性 TV
dy = f_pred[:, :, 1:, :] - f_pred[:, :, :-1, :]
dx = f_pred[:, :, :, 1:] - f_pred[:, :, :, :-1]
loss_tv = torch.mean(torch.sqrt(dx**2 + dy**2 + 1e-6))
```

1. **频谱加权的系数 MSE**（只强制“看得见”的模态）：
    Φ 在高频会变小 → 单角度下高频基本看不见；给系数 MSE 加个权重，让“可辨识”的方向更重要：

```python
phi = self.model.optimizer.operator.phi_diag      # (441,) complex
w = phi.abs().view(1, 1, 21, 21)                  # reshape 成 21×21
loss_coeff_w = torch.mean(((coeff_pred - coeff_true)**2) * w)
```

> `phi_diag` 就挂在 `RadonFourierOperator2D` 上，可以直接取用。

**综合 loss（建议超参）：**

```python
loss = (1.0 * loss_coeff_w
        + 1.0 * loss_proj
        + 0.5 * loss_img
        + 1e-3 * loss_tv)
```

把这几项合进去后，你会发现**验证损失还能往下压**，而不是一直卡在之前那条线。

# B. 数值与归一化：让单角度的数据更“好用”

1. **FFT 归一化**（避免不必要放大）：
    在 `compute_F_from_radon` 里，给 1D FFT 除以长度（N=探测器数），让尺度稳定：

```python
radon_fft = torch.fft.fft(radon_data.squeeze(1), dim=-1) / radon_data.shape[-1]
```

> 现在那块是未归一化 FFT（直接 `torch.fft.fft`），加个 `/N` 更稳。

1. **CNN 输入做按通道归一化**（不改结构也能稳）：
    在构造 `cnn_input`（你在 `model.py` 里把 `[coeff, data_grad, reg_grad, memory]` 拼一起）之后，加一行：

```python
cnn_input = torch.cat([coeff_current, data_grad, reg_grad, memory], dim=1)
cnn_input = cnn_input / (cnn_input.abs().amax(dim=(2,3), keepdim=True) + 1e-6)
```

> 你在 `get_theoretical_vs_learned_updates()` 里已经有同样的拼接位置，`forward` 里也对应。加这一步能显著减少“第一次更新过猛/后期抖动”。 

1. **限制学习更新的幅度**（单角度噪声大，强约束更稳）：
    把 `learned_update = cnn_output[:, :1] * self.step_size` 改成：

```python
learned_update = torch.tanh(cnn_output[:, :1]) * self.step_size
```

# C. 结构/训练策略：在“信息不足”里榨增益

1. **增加 unroll 步数（10 → 30/50）**
    `n_iter=10` 对单角度来说表达力偏紧（优化器太浅），建议至少 30。定义在 `LearnedGradientDescent` 初始化参数。
2. **非负性与支持域约束**

- 医学 CT 常见**非负性**：对 `f_pred` 施加 `ReLU` 或在 loss 里加 `penalty = μ*||min(f_pred,0)||²`。
- 你已经把图像限制在区域 E 外为 0（生成时），训练也可加“域外惩罚”，更一致。生成器里有 `enforce_region_constraint=True`。

1. **教师-学生（若你能合成多角度标签）**
    训练时**数据保真仍只用单角度**；但你可以离线用多角度（或 FBP）生成“teacher”图像/系数作为软监督，加入 `MSE(f_pred, f_teacher)` 或感知损失。部署时仍用单角度，不冲突。

## 为什么这些对“单角度”特别有用？

- **投影一致性 + 频谱加权**：把优化重点放在**可观测子空间**，不再被“看不见的高频系数”拖住。
- **图像/TV 先验**：单角度信息少，用先验弥补（更自然的重建就意味着更低的系数/图像误差）。
- **更多步 + 有界更新 + 归一化**：让学习优化器“更深更稳”，能啃掉平台边缘的剩余空间。
