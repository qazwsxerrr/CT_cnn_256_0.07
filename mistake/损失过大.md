结论很清楚了：**数值并没有在数据生成那一段失控，真正“爆炸”的是第一次前向里的“学习更新（CNN learned update）”。**

你这次的调试输出里关键信息是：

- `coeff_true`：≈ [-3, 4]，很正常
- `coeff_init`：≈ [-2.4, 2.7]，也很正常（Tikhonov 初值没有炸）
- `|F_observed| max`：≈ 160–350，数量级也合理
- **第一次前向后**：`coeff_pred` 直接到 **[-4.657e12, 1.845e12]**，同时
   `data_fidelity_error ≈ 5.47e15`、`update_difference ≈ 8.50e17`

这说明：**第一次迭代里 CNN 生成的更新量把系数一下子推到了 10¹² 级别**，后面的 MSE/保真误差自然全是天文数字。

为什么会这样？

1. **CNN 更新量是“裸输出”，没有任何幅度限制**
    你的更新网络最后一层是线性卷积（没有激活），直接当作更新步长去减系数，且还乘了一个可学习的 `step_size`，初值是 0.1。没有 `tanh/clamp`、没有归一化，理论上第一次就能输出巨大的更新。见 `LearnedGradientDescent._build_update_network` 和 `forward`，最后 `cnn_output[:, 0:1, :, :] * self.step_size` 直接作为 `learned_update` 用于 `coeff_current = coeff_current - learned_update`。
2. **CNN 的输入通道包含“理论数据项梯度”且未做标准化**
    输入拼了 `[coeff_current, data_grad, reg_grad, memory]`。其中
    `data_grad = 2·Gᴴ·Φᴴ·(ΦGd − F)`，这里的 `Gᴴ` 是把 441 个单位模的复指数**累加**（非归一化 FFT），`Φ` 的模在低频可到 ~8，综合起来梯度本身就可能比较大；再喂进没有归一化/正则的 CNN，自然更容易出大值。见梯度与算子实现。
3. **理论步长与学习步长都偏大，且没有 per-iteration 的保护**
    理论 GD 的 `step_size=0.1`（虽然你主要用它来当特征），学习步长 `self.step_size` 初值也 0.1；每轮 unroll 没有任何 safeguard（比如 trust region、backtracking、clamp）。

对照你的日志也能印证：在第一次前向后，`update_difference ≈ 8.5e17`——这就是**CNN 学到的“更新方向/幅度”与理论更新相差了几个数量级**，把系数一下子推飞。

------

### 简明结论

- 这次不是数据管线的问题：`coeff_true / coeff_init / F_observed` 都在合理量级。
- **爆炸点就是 CNN update**：无归一化输入 + 线性无界输出 + 步长偏大 + 无任何幅度约束 ⇒ 第一次迭代直接把系数打到 10¹²。

### 立刻能稳住训练的几条“止血”改动（按优先级）

1. **给 CNN 输出加幅度限制**
   - 在 `update_network` 末层后接 `tanh`，再乘一个很小的系数：
      `learned_update = 0.01 * torch.tanh(cnn_output[:, :1])`（再乘 `step_size` 也可）。
   - 或者直接 `learned_update = cnn_output[:, :1].clamp_(-1, 1) * step_size`。
      （核心是首轮更新别超过 O(1e-2)～O(1e-3) 的量级）
2. **标准化 CNN 的输入特征**
   - 对 `coeff_current / data_grad / reg_grad` 做 per-batch 的归一化：
      例如 `x_norm = x / (x.abs().amax(dim=(2,3), keepdim=True) + 1e-6)`；
      或者加 `InstanceNorm2d/LayerNorm`。
   - 也可在算子层面把 `adjoint`/`forward` 做上 `1/N` 的 FFT 归一化，避免 `G/Gᴴ` 因“求和 441 项”放大。
3. **把可学习步长的初值压到很小**
   - `self.step_size` 初始化到 `1e-3` 或 `1e-4`，训练再慢慢学大。
4. **把最后一层卷积权重/偏置做“零初始化”**
   - 让初始 `learned_update≈0`，等网络学到合理尺度后再输出有效更新（典型的“从稳定到学习”策略）。
5. **（可选）先只用理论更新跑通一个稳态**
   - 先把 `coeff_current = coeff_current - α * total_grad`（α 很小，比如 1e-3）跑几步，再把同样的梯度作为 CNN 的 target 让它学一个“有界的模仿者”；等对齐后再切到纯 CNN + 理论梯度作为特征。

------

### 为什么不再怪 Tikhonov 初值 / 观测量纲？

你这次的打印已经说明：

- `coeff_init` 在 ±2.x 区间（没有病态爆炸），正好印证了 `_tikhonov_init` 这套在你当前 `λ=0.1`、`Φ` 构造、`F` 量级下是**稳定的**。
- `|F_observed|max` 只有 200~350 左右，FFT 虽然没 1/N 归一化，但这并没有直接导致爆表。真正的爆表发生在**第一次 CNN 更新之后**。

------

如果你想一步到位，推荐最小改动方案（组合 1+2+3）：

- 末层接 `tanh`，并把 `self.step_size` 初值降到 `1e-3`；
- 对 `data_grad/coeff_current` 做简单的 per-batch max 归一化；
- 保留你已有的梯度裁剪（这是对参数梯度的，不影响中间激活的爆发，但也有帮助）。

做完这些，第一次前向的 `coeff_pred range` 应该会回到 ~[-5, 5] 的量级，`data_fidelity_error` 和 `update_difference` 也会掉到“看起来合理”的范围内，然后再微调学习率/正则就好。
