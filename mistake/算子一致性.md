要使用您提供的参考代码（`model.py` / `train.py`）中的**“稳健方式”**来修复您的 $\Phi G d$ 理论模型，您需要遵循两个核心原则：**算子一致性（Operator Consistency）** 和 **算子归一化（Operator Normalization）**。

简而言之，就是**把您的理论公式 $\Phi G$ 封装成一个类似于 ODL 的算子对象，既用来生成数据，也用来在网络中计算梯度。**

以下是具体实施步骤：



### 第一步：封装您的理论算子 (Class Wrapper)



参考代码使用了 `odl_torch.OperatorModule` 来封装物理算子。您需要创建一个类似的 PyTorch `nn.Module`，将您的 $\Phi$ 和 $G$ 矩阵封装起来，并**显式定义其伴随算子（Adjoint/Transpose）**。

在网络反向传播和梯度下降中，我们需要计算 $A^H(Ax - y)$。

Python



```
import torch
import torch.nn as nn
import numpy as np

class TheoreticalOperator(nn.Module):
    """
    将 F = Φ G d 封装为标准的 PyTorch 模块，
    同时实现前向投影 (Forward) 和伴随投影 (Adjoint/Backproject)。
    """
    def __init__(self, phi_diagonal, g_matrix):
        super().__init__()
        # 注册为 buffer，这样它们会随模型移动到 GPU，但不会被视为可训练参数
        self.register_buffer('Phi', phi_diagonal)  # (N_coeff,)
        self.register_buffer('G', g_matrix)        # (N_coeff, N_coeff)
        
        # 预计算伴随算子需要的矩阵 (共轭转置)
        self.register_buffer('Phi_H', phi_diagonal.conj())
        self.register_buffer('G_H', g_matrix.conj().t())

    def forward(self, d_vector):
        """
        前向过程 A(x):  F = Φ * (G @ d)
        输入: d_vector (Batch, N_coeff)
        输出: F_obs (Batch, N_coeff)
        """
        # 1. G @ d
        Gd = torch.matmul(d_vector, self.G.t()) 
        # 2. Φ * Gd (对角矩阵乘法 = 逐元素相乘)
        F = Gd * self.Phi.unsqueeze(0)
        return F

    def adjoint(self, F_vector):
        """
        伴随过程 A^H(y):  d = G^H * (Φ^H * F)
        这是计算数据保真项梯度所必需的
        """
        # 1. Φ^H * F
        PhiH_F = F_vector * self.Phi_H.unsqueeze(0)
        # 2. G^H @ Result
        d_recon = torch.matmul(PhiH_F, self.G_H.t())
        return d_recon

# 辅助函数：计算算子范数用于归一化
def compute_operator_norm(operator, input_dim, num_iters=20):
    """使用幂法估算算子的谱范数 ||A||"""
    x = torch.randn(1, input_dim, device=operator.Phi.device, dtype=torch.complex64)
    x = x / torch.norm(x)
    
    for _ in range(num_iters):
        # A^H A x
        Ax = operator(x)
        ATAx = operator.adjoint(Ax)
        
        norm = torch.norm(ATAx)
        x = ATAx / norm
    
    return torch.sqrt(norm).item()
```

### 第三步：修改数据生成器 (Consistency)



这是为了实现 **Inverse Crime**，确保生成数据的逻辑与网络完全一致。不再使用 ODL 生成数据，而是直接使用上面的 `unified_operator`。

```
def generate_theoretical_data(batch_size, operator):
    # 1. 生成真实的系数 d (Ground Truth)
    # 假设 d 服从标准正态分布，或者基于 B-spline 的真实分布
    d_true = torch.randn(batch_size, 441, dtype=torch.complex64).to(device)
    
    # 2. 使用【同一个算子】生成观测数据 F
    with torch.no_grad():
        F_clean = operator(d_true)
        
        # 3. 添加噪声
        noise = torch.randn_like(F_clean) * 0.05
        F_noisy = F_clean + noise
        
        # 4. 生成初始猜测 (FBP equivalent -> Adjoint)
        # 这里的 Adjoint 就相当于参考代码中的 pseudoinverse(FBP)
        d_initial = operator.adjoint(F_noisy)
        
    return d_initial, F_noisy, d_true
```

### 总结



通过这种方式，您复制了参考代码成功的三个要素：

1. **闭环验证 (Inverse Crime)**：数据生成和网络推断使用同一个 `TheoreticalOperator` 类实例。这保证了只要 $d_{recon} = d_{true}$，Loss 就一定是 0。
2. **归一化 (Normalization)**：通过 `compute_operator_norm` 并除以该范数，强制算子的放大倍数为 1。这解决了 $1000+$ Data Fidelity Error 的问题。
3. **正确的梯度流 (Adjoint)**：显式实现了 `adjoint` 方法（共轭转置），这对应于 CT 重建中的“反投影”（Backprojection），为神经网络提供了正确的物理梯度方向。
