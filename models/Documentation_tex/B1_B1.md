将基函数 $\Phi(x, y)$ 从当前的 $B_2(x)B_1(y)$（X轴为Hat函数/线性，Y轴为Box函数/常数）修改为 $B_1(x)B_1(y)$（双Box函数/常数，即像素基函数），意味着我们从**混合平滑度**转向了**完全分段常数（Piecewise Constant）**的模型。

以下是理论分析和具体的代码修改指南。

------



### 1. 理论分析 (Theoretical Analysis)





#### A. 空间域 (Spatial Domain)



- **当前模型 ($B_2(x)B_1(y)$)**:
  - **形状**: 在 $x$ 方向是分段线性（Hat/Triangle function，支撑区间 $[0,2]$），在 $y$ 方向是分段常数（Box function，支撑区间 $[0,1]$）。
  - **性质**: 图像在行方向（x轴）是连续的，在列方向（y轴）是不连续的。
- **修改后模型 ($B_1(x)B_1(y)$)**:
  - **形状**: 在 $x$ 和 $y$ 方向均为分段常数（Box function，支撑区间均为 $[0,1]$）。这实际上就是标准的**正方形像素（Pixel）**模型。
  - **性质**: 图像由一个个互不重叠（或仅边缘接触）的方块组成。



#### B. 傅里叶/Radon 域 (Fourier/Radon Domain)



根据傅里叶切片定理（Fourier Slice Theorem），投影的傅里叶变换等于图像傅里叶变换的切片：



$$\hat{f}(\xi) = \hat{\Phi}(\xi) \sum_{k} c_k e^{-i \langle k, \xi \rangle}$$

其中 $\hat{\Phi}(\xi_x, \xi_y)$ 是基函数的傅里叶变换。

- **定义**:
  - $\hat{B}_1(\omega) = e^{-i\omega/2} \text{sinc}(\frac{\omega}{2\pi})$ （Box的傅里叶变换是Sinc函数）
  - $\hat{B}_2(\omega) = (\hat{B}_1(\omega))^2 = e^{-i\omega} \text{sinc}^2(\frac{\omega}{2\pi})$ （Hat的傅里叶变换是Sinc平方）
- **变换**:
  - **原算子**: $\hat{\Phi}_{old}(\xi_x, \xi_y) = \hat{B}_2(\xi_x) \cdot \hat{B}_1(\xi_y) = \text{Sinc}^2(\xi_x) \cdot \text{Sinc}(\xi_y)$
  - **新算子**: $\hat{\Phi}_{new}(\xi_x, \xi_y) = \hat{B}_1(\xi_x) \cdot \hat{B}_1(\xi_y) = \text{Sinc}(\xi_x) \cdot \text{Sinc}(\xi_y)$

**影响**: 频率衰减变慢（Sinc vs Sinc平方）。高频分量会更多，这意味着在反问题中，对于高频噪声可能不如之前敏感（正则化需求可能微调），但重建出的图像会有明显的“马赛克”或阶梯状边缘。

------



### 2. 代码修改 (Code Modifications)



需要修改两个文件：`box_spline.py`（定义空间基函数）和 `radon_transform.py`（定义频域算子）。



#### 文件 1: `box_spline.py`



我们需要修改 `CardinalBSpline2D` 类，将 `phi` 函数改为两个 $B_1$ 的乘积，并更新支撑区间。

Python



```
# 修改 box_spline.py 中的 CardinalBSpline2D 类

class CardinalBSpline2D:
    """Cardinal B-spline utilities used in the main CT pipeline."""

    def __init__(self):
        # [修改点 1]: X轴支撑区间从 (0, 2) 变为 (0, 1)，因为 B1 的支撑是 1
        self.support_x = (0, 1) 
        self.support_y = (0, 1)

    # B1, B2, B1_hat_complex, B2_hat_complex 方法保持不变 ...

    def phi(self, x: np.ndarray, y: np.ndarray) -> np.ndarray:
        # [修改点 2]: 将原来的 B2(x) * B1(y) 改为 B1(x) * B1(y)
        # 原代码: return self.B2(x) * self.B1(y)
        return self.B1(x) * self.B1(y)

    # generate_cardinal_pattern 和 visualize_basis_functions 保持不变
    # 但 visualize 中 B2 的绘图说明可能不再准确对应 phi，不过不影响训练逻辑
```



#### 文件 2: `radon_transform.py`



我们需要修改 `FourierOperatorCalculator` 类，在计算 $\Phi$ 对角矩阵时，X轴方向也使用 `B1_hat_complex`。

Python



```
# 修改 radon_transform.py 中的 FourierOperatorCalculator 类

    def _compute_Phi_matrix(self):
        N = self.n_coefficients
        j = torch.arange(N, dtype=torch.float64)
        arg_x = (2.0 * np.pi * j / N) * self.beta[0].double()
        arg_y = (2.0 * np.pi * j / N) * self.beta[1].double()
        
        bspline = CardinalBSpline2D()

        # [修改点 3]: X轴方向使用 B1_hat_complex 而不是 B2_hat_complex
        # 原代码: B2 = np.asarray(bspline.B2_hat_complex(arg_x.numpy()), dtype=np.complex128)
        # 修改后:
        B1_x = np.asarray(bspline.B1_hat_complex(arg_x.numpy()), dtype=np.complex128)
        
        # Y轴保持不变
        B1_y = np.asarray(bspline.B1_hat_complex(arg_y.numpy()), dtype=np.complex128)
        
        # 计算 Phi
        # 原代码: Phi = B2 * B1
        Phi = B1_x * B1_y
        
        self.Phi_diagonal = torch.from_numpy(Phi).to(torch.complex128)
        self.xi_vector = torch.from_numpy(j.numpy().astype(np.float32))
```



### 3. 总结变化对照表



| **特性**     | **原代码 (B2(x)B1(y))**                         | **修改后 (B1(x)B1(y))**                       |
| ------------ | ----------------------------------------------- | --------------------------------------------- |
| **空间定义** | `self.B2(x) * self.B1(y)`                       | `self.B1(x) * self.B1(y)`                     |
| **支撑范围** | X: [0, 2], Y: [0, 1]                            | X: [0, 1], Y: [0, 1]                          |
| **物理含义** | X轴线性插值，Y轴常数                            | 标准像素块 (Pixel basis)                      |
| **频域公式** | $\text{sinc}^2(\xi_x) \cdot \text{sinc}(\xi_y)$ | $\text{sinc}(\xi_x) \cdot \text{sinc}(\xi_y)$ |
| **修改文件** | `box_spline.py`, `radon_transform.py`           | 同左                                          |
